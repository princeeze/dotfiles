# Core Rules

## Validation

* After any code modification, run the project's validation flow.

## Dependencies

* Install dependencies only with `pnpm i <package>`.
* Never manually add dependencies to `package.json`.
* Never use npx, prefer pnpm dlx.

## Simplicity First

* Assume new abstractions are unnecessary.
* Inline functions, service methods, handlers, variables, and utilities used called once.
* Reuse existing routers, services, utilities, and modules before creating new ones.
* If multiple functions are always called together, merge them.
* Create abstractions only when reused multiple times.

## Types

* Avoid creating new types and interfaces.
* Prefer type inference whenever possible.
* Derive types from existing sources (`typeof`, inferred returns, generated DB/API types, etc.).

## Database

* Never use `Promise.all()` inside db transactions.
* Prefer bulk insert, update, and delete ORM operations.
* Never use `.selectAll()`. Perform all filtering and aggregation in the database query.
* Avoid raw sql`` template strings or sql.raw whenever possible. Prefer standard query builder APIs.
* Do not place callbacks, `.map()`, `.filter()`, `.reduce()`, business logic, or complex computations inside ORM `.values()` or `.set()` operations. Compute first, then pass the final object.

## Frontend

* Never use arbitrary Tailwind values. Prefer built-in utility classes.

## Dates

* Use `date-fns` for all date manipulation and calculations.

## Exceptions

* Any exception must be rare, justified with an inline comment, and used only when no practical alternative exists.
